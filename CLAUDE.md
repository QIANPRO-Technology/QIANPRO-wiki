## Project Overview

QIANPRO 內部知識 Wiki，彙整 Docusaurus 資源、常用操作與工作流。雙重用途：
1. **`docs/`**（repo 根目錄）— 用繁體中文撰寫的 Docusaurus 知識文件（非網站本體）
2. **`minimal-docs-site/`** — 可直接啟動的 Docusaurus 3 參考實作站點

## Development Commands

所有 npm 指令都必須在 `minimal-docs-site/` 目錄下執行（repo 根目錄沒有 `package.json`）：

```bash
cd minimal-docs-site
npm install          # 首次安裝
npm run start        # 開發伺服器 http://localhost:3000（hot reload）
npm run build        # 輸出靜態站點到 build/
npm run serve        # 預覽 build/ 結果
npm run clear        # 清除 Docusaurus 快取（build 異常時使用）
```

## Key Configuration

### `minimal-docs-site/docusaurus.config.js`
- `future: { v4: true }` — 已啟用 Docusaurus v4 相容模式
- `onBrokenLinks: 'warn'`（非 `'throw'`）— broken link 不會中斷 build
- `baseUrl: '/'` — 根目錄部署；改為子路徑時需同步更新 Nginx location block
- `organizationName`、`projectName`、`url`、`editUrl` — 目前都是佔位符，部署前需替換

### `minimal-docs-site/sidebars.js`
目前使用 `autogenerated`（從 `docs/` 目錄結構自動產生），切換為手動排序時改為 `tutorialSidebar: [...]` 陣列。

## Deployment

Build 產出的 `build/` 是純靜態 HTML/CSS/JS，不需 Node.js runtime。

**初次部署（Ubuntu + Nginx）：**
```bash
git clone <repo> /var/www/docs-site
cd /var/www/docs-site/minimal-docs-site
npm ci
npm run build
# Nginx root 指向 /var/www/docs-site/minimal-docs-site/build
```

**更新：**
```bash
cd /var/www/docs-site/minimal-docs-site
git pull
npm ci   # 僅在 package-lock.json 有變動時需要
npm run build
```

**版本保留與回滾：** `docs/workflow/index.md` 包含使用 `/var/www/qianpro/releases/<timestamp>/` + symlink 的部署腳本範例。

**Nginx 設定：** 參考 `minimal-docs-site/deploy/nginx.conf`，子路徑部署時需同時修改 `baseUrl` 與 Nginx `location` 區塊。

## CI/CD

GitHub Actions 在 `minimal-docs-site/` 目錄下執行 build，參考 `docs/workflow/index.md` 的 YAML 範例（注意需加 `cd minimal-docs-site` 步驟）：

```yaml
- run: cd minimal-docs-site && npm ci
- run: cd minimal-docs-site && npm run build
```

## Content Management

- **靜態下載檔**：放入 `static/files/`，在 Markdown 以 `[文字](./files/xxx.pdf)` 連結
- **新增文件頁**：在 `minimal-docs-site/docs/` 建立 Markdown，front matter 用 `id`、`title`、`sidebar_label`
- **部落格文章**：在 `minimal-docs-site/blog/` 以 `YYYY-MM-DD-slug/index.md` 命名
- **自訂頁面**：在 `minimal-docs-site/src/pages/` 新增 `.js` 或 `.md`，自動對應路由

## Git Workflow

- 分支命名：`feature/<topic>`、`fix/<topic>`
- Commit 格式：`feat:`、`fix:`、`docs:`、`chore:` 語意化前綴
- 重要釋出加 tag（`v1.2.0`）
